El proyecto

Una plataforma propia
que es nuestro
caso de uso número uno.

sobremi.online nació con un objetivo doble: ser un negocio real de regalos personalizados, y ser la demostración más honesta de lo que Vacaré construye para sus clientes.

En lugar de mostrar demos o prototipos, decidimos construir algo que funciona en producción real — con usuarios reales, pagos reales y automatizaciones que corren solas todos los días.

El resultado es una plataforma donde alguien puede entrar a sobremi.online/crear, armar un regalo personalizado, pagarlo con Stripe o transferencia, y recibir toda la confirmación sin que ningún humano intervenga.

    Los desafíos técnicos
  • 1
    Manejar dos métodos de pago (Stripe y transferencia) con flujos de confirmación completamente distintos en n8n
  • 2
    Generar el link de pago de Stripe dinámicamente desde el backend con webhook de retorno
  • 3
    Registrar cada pedido en Postgres con estado actualizado en tiempo real según el webhook de Stripe
  • 4
    Construir la página pública de regalo /regalos/:id con datos dinámicos desde la base de datos
  • 5
    Enviar emails transaccionales con diseño personalizado, sin depender de plataformas de terceros

Los flujos automatizados

Todo lo que pasa
sin intervención humana.

Desde que el cliente completa el formulario hasta que el equipo recibe el aviso del pedido — todo sucede solo. Estos son los cuatro flujos n8n que corren en producción.

Flujo 01

Pago con Stripe — flujo completo

Cuando el cliente elige pagar con tarjeta, el backend de Next.js genera un Payment Intent en Stripe y devuelve el link de pago. Una vez completado el pago, Stripe envía un webhook al backend que actualiza el estado del pedido en Postgres y dispara n8n.

Flujo 02

Pago por transferencia — confirmación en dos pasos

Si el cliente elige transferencia, n8n envía inmediatamente un email de "pedido recibido, en revisión". Cuando el equipo confirma el pago manualmente, un segundo trigger de n8n envía el email de confirmación definitiva con todos los detalles del pedido.

n8n genera y envía un email con diseño personalizado que incluye los datos del pedido, el nombre del destinatario del regalo, el método de pago, el monto y el link a la página pública del regalo en /regalos/:id. Sin intervención humana. Sin plantilla genérica.

Base de datos

Postgres en VPS propio

Los pedidos se registran en una tabla con id único, nombre, producto, método de pago, monto, estado (pendiente / confirmado / entregado) y timestamps. Cada webhook de Stripe hace un UPDATE automático del estado.

-- tabla pedidos CREATE TABLE pedidos ( id UUID DEFAULT gen_random_uuid(), nombre TEXT NOT NULL, producto TEXT, monto INTEGER, metodo TEXT, estado TEXT DEFAULT 'pendiente', stripe_id TEXT, created_at TIMESTAMPTZ DEFAULT now() );
Stripe + Webhooks

Pago dinámico con confirmación automática

El backend crea un PaymentIntent de Stripe con los datos del pedido. Una vez el pago se confirma, Stripe llama al endpoint /api/stripe/webhook con el evento payment_intent.succeeded. Ese endpoint actualiza Postgres y dispara el webhook de n8n para que arranquen los flujos de email.

// api/stripe/webhook.ts const event = stripe.webhooks.constructEvent( body, sig, process.env.STRIPE_WEBHOOK_SECRET ); if (event.type === 'payment_intent.succeeded') { await updatePedido(event.data.object.id); await triggerN8N(pedidoData); }
n8n

Orquestador central

n8n 2.0 corre en Docker en el mismo VPS que Postgres. Recibe el webhook del backend, evalúa el método de pago con un nodo IF, y bifurca hacia el flujo de Stripe o el de transferencia. Cada rama genera el email y lo envía por SMTP.

Páginas dinámicas

/regalos/:id con SSR

Cada regalo tiene su propia URL pública. Next.js hace una query a Postgres en el servidor y renderiza la página con los datos del pedido. El comprador puede compartir ese link con el destinatario del regalo.

Frontend

Next.js App Router

La plataforma usa React Server Components donde puede y Client Components donde necesita interactividad (formulario de pedido, selector de productos). Deploy en Cloudflare Pages — cero costo de hosting, Edge network global.

Lo que aprendimos

Tres cosas que este proyecto
nos enseñó.

01

n8n como orquestador es imbatible para automatizaciones de negocio

Para el tipo de flujo que necesita un negocio de servicios — formulario, condicionales, emails, base de datos — n8n en un VPS propio es más potente, más barato y más flexible que cualquier herramienta SaaS de automatización. Y cuando lo conocés bien, el tiempo de desarrollo cae radicalmente.

02

Los webhooks de Stripe son la pieza más crítica — y la más subestimada

El 80% de los problemas de ecommerce que vemos en clientes tienen que ver con pagos que "quedaron colgados" sin confirmación. La firma criptográfica del webhook de Stripe y el manejo correcto de reintentos es lo que hace que el sistema sea confiable en producción real, no solo en demos.

03

Next.js + Cloudflare Pages es la combinación ganadora para la mayoría de los proyectos

Cero costo de hosting, Edge network global, Core Web Vitals al 100 de fábrica. Para proyectos que no requieren un servidor persistente (que es la mayoría), esta combinación elimina la mayor excusa para no tener una web rápida y profesional.
